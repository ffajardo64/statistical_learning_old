Geração de números pseudo-aleatórios¶

Geradores de números pseudo-aleatórios¶

  • A gerao de eventos de variveis aleatrias independentes e identicamente distribuidas com distribuio uniforme em $(0,1)$ essencial para construir eventos de variveis aleatrias com outras distribuies e/ou outras estruturas de dependncia;

Distribuio uniforme



\[ \begin{aligned} f(u)&=\begin{cases}1, & \text{ para } u\in(0,1);\\ 0, & \text{ em caso contrrio}\end{cases}\\ F(u)&=\mathbb{P}\left[U\le u\right]=u\\ \mathbb{E}[U]&=\frac12 \qquad \mathrm{var}[U]=\frac{1}{12} \end{aligned} \]

Algoritmos¶

As duas propriedades bsicas de um algoritmo $f:\mathbb{R}^k\longrightarrow\mathbb{R}$ capaz de gerar sequncias pseudo-aleatrias so

  1. A funo $u_i = f(u_{i1},\ldots, u_{ik})$ deve ser de custo computacional relativamente baixo;
  2. Deve ser difcil conhecer $u_{i1}$ dado s o conhecimento dos valores $(u_i, u_{i2},\ldots, u_{ik})$.

Outras propriedades desejveis para um gerador deste tipo de sequncias so a repetibilidade, a portabilidade, a periodicidade.

Geradores congruenciais lineares¶

Geradores congruenciais¶

Uma classe interessante de geradores, conhecida como congruenciais lineares, definida pela recurso

$$u_i \equiv (a\,u_{i1} + c)\mod M,$$

onde $i=1,2,\ldots$, $u_0$ chamada de semente, $M$ chamado de mdulo, $a$ o multiplicador e $c$ o incremento. O operador $\equiv$ indica a aritmtica modular entre os nmeros.

Se o incremento nulo trata-se de um gerador misto, caso contrrio de um gerador multiplicativo.

# Gerador congruencial linear

lcg <- function(n,a=1664525,c=1013904223,m=2 ** 32){
    
    rng <- numeric(n)
    
    seed <- as.numeric(Sys.time()) * 1000 # Semente em funo do relgio do computador
    
    for (i in 1:n){
        seed <- (a * seed + c) %% m
        rng[i] <- seed / m
    }
    return(rng)
}

Geradores com recursão múltipla¶

Esse tipo de geradores so baseados na seguinte forma funcional:

$$x_n \equiv (a_1 x_{n1} + \cdots + a_k x_{nk})\mod M,$$

onde $k$ um inteiro fixo. Para calcular os valores no intervalo $(0,1)$ calcule $u_n=\frac{x_n}{M}$.

Geradores congruenciais combinados¶

Esse tipo de geradores so baseados na seguinte forma funcional:

$$x_i \equiv \Bigg(\sum^k_{j=1} (-1)^{j-1} y_{i,j} \Bigg) \mod (M 1)),$$

onde $M$ o mdulo, $y_{i,j}$ o $i$-simo elemento do $j$-simo gerador congruencial linear. Para calcular os valores no intervalo $(0,1)$ calcule $u_n=\frac{x_n}{M}$.

# Um exemplo de um gerador congruencial combinando dois geradores congruenciais lineares.
comb.2lcg <- function(n=100, a1 = 40014, m1 = 2147483563, a2 = 40692, m2 = 2147483399){
    rng <- vector(length = n)
    
    y1 <- runif(1, 1, m1 - 1) # Sementes
    y2 <- runif(1, 1, m2 - 1)
    for (i in 1:n){
        y1 <- a1 * y1 %% m1
        y2 <- a2 * y2 %% m2
        x <- (y1 - y2) %% (m1 - 1)
        if (x > 0){
            rng[i] <- x / m1
        } else if (x < 0){
            rng[i] <- (x / m1) + 1
        } else if (x == 0){
            rng[i] <- (m1 - 1) / m1
        }
    }
    return(rng)
}

comb.3lcg <- function(n=100, a1 = 157, m1 = 32363, a2 = 146, m2 = 31727, a3 = 142, m3 = 31657){
    rng <- vector(length = n)
    # Sementes
    y1 <- runif(1, 1, m1 - 1)
    y2 <- runif(1, 1, m2 - 1)
    y3 <- runif(1, 1, m3 - 1)
    for (i in 1:n){
        y1 <- a1 * y1 %% m1
        y2 <- a2 * y2 %% m2
        y3 <- a3 * y3 %% m3
        x <- (y1 - y2 - y3) %% (m1 - 1)
        if (x > 0){
            rng[i] <- x / m1
        } else if (x < 0){
            rng[i] <- (x / m1) + 1
        } else if (x == 0){
            rng[i] <- (m1 - 1) / m1
        }
    }
    return(rng)
}

# Gerador de Wichmann-Hill de 1982

wh.predict <- function(x){
    M1 <- 30269
    M2 <- 30307
    M3 <- 30323
    y <- round(M1*M2*M3*x)
    s1 <- y %% M1
    s2 <- y %% M2
    s3 <- y %% M3
    s1 <- (171*26478*s1) %% M1
    s2 <- (172*26070*s2) %% M2
    s3 <- (170*8037*s3) %% M3
    (s1/M1 + s2/M2 + s3/M3) %% 1
}

Verificando qualidade dos geradores¶

"Qualidade dos geradores" faz referncia se o conjunto de valores gerados satisfazem as propriedades bsicas:

  1. A sequncia de valores so realizaes de uma amostra aleatria de variveis uniformes no intervalo $(0,1)$;
  2. No possvel conhecer $_{1}$ dado s o conhecimento dos outros valores da sequncia.

Verificando qualidade dos geradores¶

Um usurio cuidadoso de plataformas computacionais que utiliza simulao estocstica deveria verificar a qualidade dos geradores disponveis por meio de testes estatsticos, por exemplo:

  1. Avaliao qualitativa: grficos para avaliar a aderncia distribuio uniforme, dependncia seqencial e ausncia de vazios no espao;
  2. Testes de aderncia gerais: $\chi^2$ , Kolmogorov-Smirnov etc;
  3. Testes de aderncia especficos: serial, do intervalo, pquer, do mximo etc;
  4. Testes de independncia: de permutaes, de dependncia linear.

pi.approx<-function(n=100, plot=TRUE){
    x<-lcg(n)
    y<-lcg(n)
    d<-(x^2+y^2<1)
    pi.approx<-4*(sum(d)/n)
    if(plot){
        obj = list(n=n, pi=pi.approx)
        plot(x,y,col=d+2,pch=20, main=bquote(paste("n" == .(obj$n),", ",pi==.(obj$pi))),cex.main=1.5)
        curve(sqrt(1-x^2), 0, 1,add=TRUE, col="blue")    
    }
}
